# TeamQueue

과제 공지를 입력하면 팀 역할과 작업 큐로 정리해 주고, 팀플 배분과 진행 상황까지 한 번에 관리할 수 있는 학생용 웹 서비스입니다.

이 프로젝트는 팀플에서 자주 생기는 문제를 줄이기 위해 만들었습니다. 공지가 길어서 읽기 어렵거나, 누가 어떤 일을 맡을지 정하기 애매하거나, 배정한 뒤 진행 상황을 놓치는 상황을 줄이는 것이 목표입니다.

## 제안서용 정리

### 응용 분야

학생 팀플 과제 지원 웹 서비스입니다. 교수님 공지, LMS 안내문, 팀플 요구사항을 입력하면 팀 역할과 작업 큐로 바꾸고, 배정안과 진행 상황을 함께 관리합니다.

   - 스크린샷 정리, 시연 시나리오 작성, PPT와 최종 설명 문구를 다듬습니다.

### 분야 선정 이유

Queue는 `먼저 들어온 일을 먼저 처리하는` 자료구조이기 때문에, 과제 공지처럼 순서대로 정리해야 하는 정보를 다루는 데 잘 맞습니다. 팀플에서는 해야 할 일이 한꺼번에 들어오고, 확인이 필요한 항목과 배정할 항목이 계속 늘어나므로, 이를 FIFO 방식으로 차례대로 관리하면 흐름이 명확해집니다. 또한 조원별 역할과 가능 시간을 고려해 업무를 한 번에 정리할 수 있어, 공정한 배분과 진행 관리에도 도움이 됩니다.

### 개발을 통한 기대

이 프로그램을 통해 팀장은 과제 공지를 다시 읽으며 정리하는 시간을 줄일 수 있고, 팀원들은 각자 맡을 일을 더 빠르게 이해할 수 있습니다. 배정이 끝난 뒤에도 진행 상황을 계속 추적할 수 있어서, 팀플에서 자주 생기는 업무 누락이나 역할 편중을 줄일 수 있습니다. 결과적으로 과제 해석, 역할 분담, 진행 관리까지 하나의 흐름으로 연결되는 것이 기대됩니다.

### Queue 적용 사항

- 과제 공지에서 추출한 작업을 `작업 큐`에 넣고 순서대로 검토합니다.
- 보충 정보가 부족한 항목은 `질문 큐`에 넣어 다시 확인합니다.
- 수정이 필요한 배정안은 `검토 큐`에서 다시 조정합니다.
- 마감이 임박했거나 멈춘 작업은 `보류/재배정 큐`로 따로 관리합니다.
- 확정된 작업의 상태 변화는 `모니터링 큐`와 기록 목록으로 이어집니다.

Queue는 단순히 줄 세우기만 하는 것이 아니라, `수집 -> 분석 -> 확인 -> 배정 -> 추적` 과정을 차례대로 관리하는 핵심 구조로 사용됩니다.

## OpenAI API 키 설정

OpenAI 분석을 사용하려면 개인 API 키를 환경변수로 넣어야 합니다. 키는 GitHub에 올리면 안 되므로, 저장소 안에는 `README.md`와 `screenshots/`만 들어가고 비밀값 파일은 로컬에만 둡니다.

### 1) API 키 발급

OpenAI 계정에서 API key를 새로 발급받습니다.

### 2) 로컬 설정 파일 만들기

프로젝트 폴더에서 `.env.example`을 복사해서 `.env.local` 파일을 만듭니다.

```bash
cp .env.example .env.local
```

### 3) 키 입력

`.env.local` 파일을 열고 `OPENAI_API_KEY`에 본인 키를 넣습니다.

```bash
OPENAI_API_KEY=sk-...
PORT=3000
```

주의할 점:

- 키는 한 줄로만 넣습니다.
- 앞뒤에 공백을 넣지 않습니다.
- 주석이나 한글 설명을 같은 줄에 붙이지 않습니다.
- 키를 다른 사람과 공유하지 않습니다.

### 4) 서버 다시 실행

키를 수정한 뒤에는 서버를 껐다가 다시 켜야 합니다.

```bash
npm start
```

## OpenAI 키가 없을 때는 어떻게 되나

OpenAI 키가 없거나, 키가 잘못되었거나, API 호출이 실패하더라도 앱은 계속 동작합니다.

동작 순서는 다음과 같습니다.

1. OpenAI 키가 있으면 먼저 OpenAI로 분석합니다.
2. OpenAI 호출이 실패하면 로컬 C 분석 엔진 `teamqueue_engine.c`로 다시 시도합니다.
3. C 엔진도 실패하면 JavaScript 기반의 로컬 규칙 분석으로 마지막 폴백을 수행합니다.

즉, 인터넷이 없거나 API가 잠깐 불안정해도 기본 기능은 계속 사용할 수 있습니다. 다만 성능 차이가 많이 나니, 어지간하면 openAI에서 테스트용 키 하나 발급받으셔서 넣어보는 걸 추천드립니다!

## 로컬 C 분석 엔진

이 프로젝트에는 자료구조 수업 취지에 맞게 C로 만든 로컬 분석 엔진이 포함되어 있습니다.

- 파일: `teamqueue_engine.c`
- 실행 파일: `teamqueue_engine`
- 역할: 과제 텍스트를 바탕으로 작업 목록, 질문 목록, 추천 배정안을 만드는 로컬 분석 보조기

서버가 처음 실행될 때 C 컴파일러가 있으면 자동으로 빌드됩니다. macOS에서는 보통 기본 `cc` 또는 Xcode Command Line Tools가 있으면 됩니다. 만약 C 컴파일러가 없더라도 앱은 JavaScript 폴백으로 계속 실행됩니다.
